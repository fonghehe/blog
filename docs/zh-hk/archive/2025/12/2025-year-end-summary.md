第八篇年終總結。今年的主題詞是：**平衡**。

## 2025 年的前端技術

總體感覺：框架趨於穩定，工具鏈大幅改進，AI 能力躍升。

**主要變化：**

```
React 19 正式版（年初）：Actions API 生產可用
TypeScript 5.7/5.8：持續小步迭代
Tailwind CSS 4 正式版：Oxide 引擎，CSS 配置
Vite 6：Rolldown（Rust 編譯器）正式合併
Bun 2：Node.js 兼容性更好
CSS 原生：container queries、:has()、@layer 普及

AI 工具：
  Claude 3.7 / GPT-5：更強的代碼理解
  Cursor 成為主流編輯器
  AI Code Review 工具開始進入工作流
```

**沒有發生的：**

```
Web Components 還是沒有主流化
WebAssembly 在前端業務層還是沒有普及
Deno 市場份額依然很小
```

## 今年我做的事

**技術層面**：完成了團隊的設計系統建設，從 Figma token 到代碼 CSS 變量的全鏈路打通。這是今年最有成就感的事情，雖然過程很痛苦（主要是協調問題，不是技術問題）。

**領導力層面**：今年開始承擔更多 tech lead 工作。做的決定越來越多，寫的代碼越來越少。這個轉變有點適應困難，需要接受"影響力不一定來自親自寫代碼"。

**關於 AI**：今年有個重要的認知轉變。

去年我把 AI 當"代碼生成器"，今年我開始把 AI 當"思考夥伴"。區別是：

```
代碼生成器：告訴 AI 要寫什麼，AI 輸出代碼
思考夥伴：把問題告訴 AI，和 AI 一起思考解法

前者省的是打字時間
後者省的是思考時間（但不是替代思考）
```

## AI 和工程師的新平衡

今年觀察到一個有趣的現象：

使用 AI 最好的工程師，往往是經驗最豐富的那些。因為他們：

- 知道 AI 容易在哪裏犯錯
- 能快速判斷 AI 的輸出質量
- 知道哪類問題 AI 擅長，哪類不擅長
- 有足夠的上下文把問題描述清楚

而經驗少的工程師，有時會把 AI 的輸出當成答案，而不是草稿。

這讓我對"AI 會替代程序員"的論斷更加懷疑。AI 是放大器，它把好工程師變得更好，但也可能把不好的工程習慣放大。

## 給年輕工程師的建議

今年我帶了兩個新人，有一些感受。

別隻學框架：

```
React/Vue 隻是工具，瀏覽器、HTTP、數據結構、算法才是基礎。
基礎不牢，框架換了就得重學。
```

理解再使用：

```
AI 生成了一段代碼，如果你不理解它，就不要提交。
不是因為 AI 會出錯（雖然它會），
而是因為你要對這段代碼負責。
```

積累判斷力：

```
判斷力來自經驗。經驗來自犯錯。
不要怕犯錯，要怕不從錯誤裏學習。
```

## 2026 展望

技術上，我有幾個預判：

```
React 生態：Server Components 成為主流，傳統 SPA 減少
AI 工具：從輔助編碼到輔助設計，會提前進入 AI 協同的工作流
CSS：原生能力繼續增強，Tailwind 之類的工具可能被重新審視
打包工具：Rust-based 工具（Rolldown、Biome）全面成熟
```

個人層面，我想：

```
- 在開源社區做更多貢獻
- 學習更多系統設計（不侷限於前端）
- 寫作：更多關於工程文化、團隊協作的內容
- 找到技術深度和廣度的新平衡
```

## 小結

八年前端，從學習者到實踐者到（某種程度上的）領導者。每個階段都有不同的困惑和成長。

2025 年最大的收穫：技術能力不隻是寫代碼的能力，也包括判斷什麼代碼值得寫，以及如何讓團隊寫出好代碼。

明年見。
