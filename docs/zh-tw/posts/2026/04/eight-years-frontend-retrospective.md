今年是我做前端的第八年。不是年終總結，但想借這個時間節點梳理一下：一個前端工程師的成長路徑是什麼樣的，哪些事情是真正重要的。

## 我的時間線

```
2017：初級前端。jQuery、CSS、偶爾用 Vue
2018：開始寫這個部落格。Webpack、Babel、ES6+ 成了日常
2019：React 和 TypeScript 成為主力。開始關注工程化
2020：疫情居家。大量學習時間。Vue 3、Vite 早期使用者
2021：第一次獨立負責較複雜的前端模組
2022：開始做技術分享。Next.js、React 架構深入
2023：AI 工具元年。開始思考"工程師是什麼"
2024：承擔更多技術決策。設計系統建設
2025：正式擔任 tech lead。技術和管理的平衡
2026：現在
```

## 哪些事情真正重要

**1. 基礎比框架重要**

我在 2018 年寫了很多 Vue、Webpack 的文章，但這些東西現在都換了。真正沒換的是：

```
- HTTP：快取、請求最佳化、狀態碼
- 瀏覽器渲染：CRP、重排重繪、合成層
- JavaScript 基礎：原型鏈、Event Loop、閉包
- 演算法和資料結構：不用每道 hard，但基礎要紮實
- 設計模式：不用死記，但遇到問題時能認出來
```

框架知識有保質期，基礎沒有。

**2. 寫作是最好的學習方式**

這八年寫了 100+ 篇部落格，每篇文章都迫使我把模糊的理解變成清晰的表達。很多"我以為我懂了"的事情，寫的時候才發現沒懂。

如果你只能做一件事來提升自己，就是寫作。

**3. 溝通能力比技術能力更稀缺**

工作三年內，技術能力是主要變數。三年之後，溝通和影響力越來越重要。

能把技術決策講清楚給非技術人員聽，能讓團隊接受你的方案，能在 code review 裡讓別人心服口服——這些比寫出多優雅的程式碼更難，也更值錢。

**4. 選擇"難"的事**

每次技術選型，我會問：哪個方案能讓我學到更多？

當然不能忽視專案需求，但在條件允許時，主動選擇有學習價值的方案。不要總待在舒適區。

## 關於 AI 工具的長遠看法

我在 2023 年開始認真使用 AI 工具，到現在兩三年。有幾個觀察：

**AI 提高了"寫程式碼"的速度，但沒有減少"思考設計"的需求**

需求分析、架構設計、權衡取捨——這些 AI 還是做不好。可能不是因為 AI 不夠聰明，而是因為這些判斷高度依賴上下文，包括公司文化、團隊能力、歷史債務等等。

**AI 讓"平均水平"更容易達到**

任何人用 AI 都能寫出中等水平的程式碼。那工程師的差異體現在哪？我認為是：識別問題的能力（知道要解決什麼）和質量判斷能力（知道什麼是好的解法）。

**結論：繼續學，不要焦慮**

技術在變，但工程師的核心價值沒有變：理解問題，設計方案，交付結果。

## 給八年前的自己說

如果能回到 2018 年，我會告訴那時的自己：

```
- 不要追逐每一個新框架，選一個學深
- 多寫文章，把學到的東西寫出來
- 學英語，把英語做好，文件/社群會開啟很多門
- 參與開源專案，哪怕只是提 issue
- 和比你厲害的人一起工作，這是最快的成長方式
```

## 現在做什麼

今年開始貢獻一個開源的 React 元件庫（還在早期）。做了一些技術分享。團隊裡培養了兩個不錯的工程師。

沒有宏大的目標，只想繼續做一個靠譜的工程師。
